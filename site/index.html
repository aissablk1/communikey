Bus de messages · signé, chiffré E2E · post-quantique · souverain

Tes agents de code, sur le même fil.

csend injecte un message dans une autre session d’agent CLI déjà en cours — en lisant son état avant d’agir (idle / busy / confirmation). Signé (tu sais qui parle, pas juste « quelqu’un avec le bon secret »), chiffré pour son destinataire, post-quantique, souverain, cross-provider, cross-OS. Go, zéro dépendance, Apache-2.0.

01 — Le trou que personne ne comble

Ce que csend fait — seul.

Aucune API officielle n’injecte un prompt dans une session d’agent CLI vivante. Les orchestrateurs lancent leurs agents ; tmux tape à l’aveugle ; aucun ne parle, chiffré, à une session déjà ouverte ailleurs. csend est ce fil-là.

02 — Modèle de sécurité

Casser les deux serrures pour lire.

Primitives auditées de la stdlib Go 1.24 — jamais de crypto maison. Trois garanties, expliquées simplement.

E2E

Chiffré de bout en bout

Chaque message est signé Ed25519 puis chiffré sous un secret X25519 ⊕ ML-KEM-768 et scellé en AES-256-GCM. Seul le destinataire lit ; le journal ne garde qu’un hash, jamais le clair.

  • Ed25519
  • X25519
  • AES-256-GCM
PQC

Hybride post-quantique

Le secret combine un KEM classique et un KEM à réseaux euclidiens (ML-KEM-768). Il faut briser les deux pour déchiffrer — la parade au « Harvest Now, Decrypt Later », qui capte aujourd’hui pour casser demain.

X25519 ⊕ ML-KEM-768
N/M

Vault + recovery par seuil

L’identité vit dans un vault chiffré AES-256-GCM. La clé se découpe en parts Shamir N-sur-M sur GF(2⁸) : K-1 parts ne révèlent rien. Perds une part, garde l’accès. 

$ csend recovery split 2 3
✓ 3 parts · seuil 2-sur-3
$ csend recovery combine p1 p2
✓ identité reconstituée

03 — Quickstart

Du go install au premier message.

  1. 1

    Installe

    $ go install github.com/aissablk1/csend@latest

    Binaire universel arm64 + x86_64. Aucune dépendance à résoudre.

  2. 2

    Voie coopérative — marche partout

    $ csend inbox SACEM "lance le build de prod"
$ csend recv                  # l’autre session relève

    L’inbox coopératif ne dépend d’aucun multiplexeur : tout OS, tout provider.

  3. 3

    Voie cmux — pilotage state-aware

    $ csend list                  # états live
$ csend tree                  # graphe père → enfants
$ csend send SACEM "go"      # validé seulement si idle

    Par défaut --auto : valide si idle, dépose si busy, refuse sur un prompt y/N.

  4. 4

    Identité + recovery

    $ CSEND_VAULT_PASS=… csend id --create
$ csend recovery split 2 3     # seuil 2-sur-3

    Exposé aussi comme skill Claude Code : « dis à SACEM de… ».

04 — État réel

Honnête sur ce qui marche.

csend est en alpha. Voici la frontière exacte entre le livré et le prévu — sans flou.

Livré & testé

  • Inbox coopératif (universel)
  • Injection cmux + tmux state-aware
  • Adaptateurs Codex + Gemini (calibrés sur source)
  • Graphe familial père → enfants
  • Crypto E2E hybride PQC + vault
  • Recovery Shamir N-sur-M
  • Mémoire / journal interrogeable

En route

  • Backend screen / ConPTY
  • Vault à passkey (WebAuthn)
  • Réseau multi-machine — TLS hybride PQC
  • Recovery en phrases (BIP-39)
  • Mobile · Windows natif · bridge Agent Teams

Windows natif et mobile resteront coopératif-only : pas d’injection clavier. On le dit avant, pas après.
